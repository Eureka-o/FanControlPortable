# FanControl 产品设计与可维护性优化清单

## 背景

本轮优化基于 Product Design 审查结果，目标不是重做界面，而是在现有 FanControl 桌面软件风格下继续打磨：保持左侧导航、Mica/系统主题、设置页行式结构、中文优先文案和信息密度，同时降低后续功能迭代时的代码风险。

## 设计准则

- 继承现有设计元素：设置项、折叠菜单、按钮、卡片、边框、主题变量和图标风格都优先复用现有实现。
- 面向电脑端软件，不按手机应用重新设计；桌面信息密度可以高，但必须清楚、可扫读。
- 不新增割裂的颜色体系，不硬编码无法随主题切换的颜色。
- 新增功能入口要和设备能力绑定，不能让不支持的设备看到无效功能。
- WiFi、虚拟串口、BLE/HID、飞智设备和 DIY 设备的 UI 入口要保持边界清晰。
- 说明文案中文优先，`Beta` 这类产品状态标识可以保留英文。

## 工程准则

- 高内聚、低耦合：展示组件、设备档案辅助、协议/发包、扫描、控制策略、诊断日志分别放在合适模块里。
- `ControlPanel.tsx` 不再继续承担所有函数；优先抽出纯 UI primitives 和无副作用 helper。
- 后续新增设备功能时，先定义 capability/profile，再让前端根据能力自适应显示。
- 性能敏感逻辑避免在每次 render 中反复创建大数组、重复过滤或重复格式化。
- 兼容旧配置：迁移字段、旧 IP、曲线、设备档案、托盘和诊断路径不能因为 UI 整理丢失。

## 代码注释习惯

- 对复杂状态机、协议边界、兼容迁移、性能优化点加短注释。
- 注释解释“为什么这样做”，不要解释显而易见的代码。
- 不为临时 UI 样式堆注释；样式应通过命名、结构和复用组件保持可读。
