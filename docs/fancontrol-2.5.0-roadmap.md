# FanControl 2.5.0+ 优化与增强清单

> 状态：规划清单。以下项目均为后续优化方向，不代表已经实现。
> 目标：保留 THRM 主题致敬，同时把差异化集中在状态可信度、安全控制、设备配置和运行效率上。

## 最高优先级

- [ ] **设备状态机重做**
  - 统一 BLE、HID、WiFi、串口的状态语义：发现中、已连接、能力读取中、可控制、暂时不可用。
  - 页面明确区分“已连接”和“已经可以控制”，避免连接成功但实际写入失败。
  - 状态卡显示当前阶段、最后一次成功通信时间和下一步可执行操作。

- [ ] **安全控制兜底**
  - 覆盖断连、睡眠恢复、设备切换、温度传感器失效、核心服务异常等路径。
  - 控制失败时保留安全转速或安全模式；恢复成功后再交还自动控制。
  - 防止旧设备、旧温度样本和旧目标在切换或恢复后继续写入。

- [ ] **设备配置中心**
  - 每个设备独立保存风扇曲线、手动档位、温度来源和 SmartControl 参数。
  - 增加静音、办公、游戏、性能等预设，并支持临时增强后自动恢复。
  - 切换预设前显示受影响项目和冲突项，避免静默覆盖用户配置。

- [ ] **SmartControl 可解释化**
  - 在状态卡直接说明当前动作原因，例如“因 CPU 功耗上升提前增速”或“因温度稳定正在降速”。
  - 显示输入是否新鲜、当前判断是否处于保护模式；不新增日志中心。
  - 用户可以一键暂停 SmartControl 并恢复原有自动控制。

## 性能优化

- [ ] **按需轮询设备**：只启动当前需要的传感器和设备轮询，未使用的传感器不持续读取。
- [ ] **合并配置写入**：对连续设置变更做去抖和合并，避免频繁写文件或重复下发设备命令。
- [ ] **局部更新监控界面**：数据变化只刷新相关卡片，减少整页重渲染和动画重复播放。
- [ ] **历史曲线采样压缩**：限制长期运行时的采样数量，保证跨天查看不会因数据增长而卡顿。
- [ ] **扫描、连接、能力读取异步化**：设备探测不能阻塞 UI；启动时先显示轻量骨架，再补充详细能力。
- [ ] **设定性能指标**：前台交互不因设备扫描阻塞；控制循环保持稳定周期；空闲状态降低传感器读取频率。

## 可形成差异化的功能

- [ ] **热管理预设**：静音、平衡、性能、极限四种模式，支持临时增强和自动恢复。
- [ ] **温度与噪音校准向导**：引导用户记录不同转速下的温度和噪音，生成更适合本机的曲线建议。
- [ ] **设备能力面板**：明确展示当前设备支持的 RPM、百分比、RGB、休眠归零、自动扫描等能力。
- [ ] **托盘快速控制**：直接切换预设、暂停 SmartControl、恢复自动控制，不必打开主窗口。
- [ ] **传感器可信度提示**：区分实时、延迟、不可用数据，并在控制决策受到影响时给出简短提示。

## 美观度与信息层级

- [ ] **首页首屏聚焦**：优先突出当前温度、风扇状态、控制模式和设备状态。
- [ ] **毛玻璃收敛到容器**：窗口和主要容器保留材质效果，普通小行不全部透明化。
- [ ] **统一视觉节奏**：统一卡片间距、标题层级、数字字号和状态颜色。
- [ ] **统一页面动效**：所有页面采用同一套进入、退出、展开动效，并尊重减少动态效果设置。
- [ ] **设置页密度优化**：使用“标题 + 当前值 + 一个操作”的结构，减少大段说明文字。

## 建议实施顺序

1. 设备状态机与安全兜底。
2. 设备配置中心与预设切换。
3. SmartControl 可解释化。
4. 按需轮询、异步扫描和历史数据性能优化。
5. 托盘快速控制、校准向导和能力面板。
6. 最后统一信息层级与页面视觉细节。

## 明确暂不纳入

- [ ] 日志中心或复杂日志浏览器。
- [ ] 删除现有 THRM 致敬主题或强行改成完全不同的视觉语言。
- [ ] 为了装饰而增加大面积透明、渐变或高频动效。
