# FanControl 2.5.2

2.5.2 汇总了三个预览版本的改进，重点提升 WiFi 自动连接、休眠恢复、历史趋势操作和软件内更新体验，并进一步缩小安装包体积。

## 注意

- 本版本调整了 WiFi、HID 和休眠后的设备恢复流程。不同设备和网络环境可能存在差异；如遇到无法连接、状态不同步或自动控制异常，建议导出诊断信息并附上设备型号、连接方式和复现步骤。
- 动态 IP 自动修复仅在用户启用该功能后生效，并且只会在明确发现一台新设备时更新地址，避免多设备环境下误连。

## 更新亮点

- **WiFi 自动连接**：开启 WiFi 兼容模式后，软件启动和重新连接时会自动尝试已保存的设备，不再需要首次进入软件后手动点击连接。动态 IP 变化时可以自动发现新地址并重试。
- **更可靠的设备恢复**：系统从睡眠或休眠恢复后会等待设备重新就绪并分阶段重连；飞智 HID 设备重新出现时也会立即恢复连接，旧连接数据不会覆盖当前状态。
- **更安全的自动控制**：温度数据连续不可用时，会临时使用当前设备可执行的安全转速；温度恢复后自动返回正常曲线控制。手动断开的设备不会在唤醒后自行重连。
- **历史趋势双图展示**：功耗与温度、风扇趋势分成上下两个对齐图表。悬浮任一图表都会显示该时刻全部已启用数据；取消所有功耗项后，下方图表自动隐藏。
- **曲线页操作更顺畅**：历史图接近可视区域时才会加载，切换页面不再重复获取历史记录；从首页进入曲线编辑或历史详情时，定位不会再因后续布局变化而偏移。
- **软件内更新更完整**：支持安装包校验、断点续传、暂停、继续、中止和失败重试；代理连接失败时会尝试直连，切换页面不会中断当前下载。
- **更轻的安装包**：精简重复的内置中文字体资源，在保持现有界面字体风格和中文显示兼容性的同时，减少安装包与便携包体积。

## 修复与改进

- 修复下载浮窗收起为进度圆环后，部分情况下无法重新展开的问题；收起态和展开态均可拖动。
- 下载阶段、错误状态和重试次数现在与实际下载任务保持一致，开始下载和失败重试时的反馈更及时。
- 统一正式版、Preview 和 nightly 的版本比较，旧预览版可以正确识别后续预览版和本次正式版。
- 自动清理无效断点、旧版临时更新文件和超过 7 天的已知更新残留，同时保留近期断点和无关文件。
- 配置保存失败时会保留原配置，并明确提示操作失败，避免界面显示成功但设置并未写入。
- WiFi 自动连接后，连接方式、地址、实时转速和设备能力会同步显示，自动控制可以直接进入正常状态。
- 历史趋势新增休眠、断开、重连和曲线方案切换标记，便于对照设备状态变化。
- 设置页分区切换时保留当前滚动位置，不再被强制拉回顶部。
- 设备断开或温度读取异常时提供排查指引，并保留重试、重启温度服务和导出诊断信息入口。

支持从旧版本直接升级；已有配置、设备档案、曲线方案、学习数据、历史记录、主题、托盘设置和 IP 设置会继续保留。
