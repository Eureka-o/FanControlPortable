# FanControl 2.5.2 Preview 3

这是 2.5.2 的第三版功能预览，重点优化休眠后的设备恢复、自动控制安全性、历史趋势展示，以及更新下载过程中的状态反馈。

## 连接与休眠恢复

- 系统从睡眠或休眠恢复后，会优先重连上次使用的原生设备档案，并使用分阶段重试降低设备尚未就绪时的连接失败。
- 飞智 HID 设备重新出现时会立即触发连接恢复；旧读取任务的数据不会再覆盖新连接状态。
- 设备断开和温度读取异常时会显示分步排查指引，并保留重试、重启温度服务和导出诊断信息的入口。

## 自动控制保护

- 自动控制连续多次拿不到可信温度时，会临时切换到当前曲线允许的安全转速；温度恢复后自动回到正常曲线控制。
- 安全转速仍会遵守飞智设备当前供电和挡位能力上限，避免下发设备无法执行的目标。

## 历史趋势与页面操作

- 功耗趋势与温度、风扇趋势分成上下两个图表，并保持左右绘图区对齐。
- 鼠标悬浮在任一图表时都会显示该时刻所有已启用的温度、风扇和功耗信息；取消所有功耗项后，下方功耗图会自动隐藏。
- 历史趋势会标记休眠、断开、重连和曲线方案切换事件；功耗图延迟到接近可视区域时再加载，减少进入曲线页时的卡顿。
- 设置页三个分区之间切换时保留当前滚动位置，不再被强制拉回顶部。

## 配置保存与控制反馈

- 配置写入失败时会保留原有配置，避免出现界面已经改变、磁盘配置却未保存的不一致状态。
- 挡位灯、通电启动、智能启停、WiFi 待机转速、亮度和 Windows 自启动等操作在保存失败时会返回明确结果，不再广播为已成功保存。

## 更新检测

- 统一正式版、Preview 和 nightly 的版本比较规则，Preview 2 可以正确识别 Preview 3，后续同系列预览版本也会按修订号排序。
- 手动检查更新仍会在已是最新版时给出原生提醒，发现新版本时显示对应版本号。

## 下载进度

- 下载阶段、暂停、继续、取消、安装和错误状态统一以后端事件为准，减少网络响应期间的状态跳变。
- 重试次数由实际下载策略提供，进度浮窗不再使用固定的前端重试上限。
- 开始下载时会更快显示初始进度，切换页面后仍能保持当前任务状态。
- 下载浮窗折叠后可以重新展开，并继续显示当前下载状态。

此版本为预览版，建议重点验证休眠后的 HID/WiFi 重连、温度读取中断时的安全转速、历史趋势双图交互，以及下载浮窗的折叠和恢复。
