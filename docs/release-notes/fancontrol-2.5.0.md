# FanControl 2.5.0

相比 2.4.4，本次更新重点改善曲线方案管理、设置页面、传感器策略、休眠恢复和整体操作体验。

## 曲线方案

- 重新设计曲线方案管理区域，新增、选择、重命名和删除更加方便。
- 导入曲线时自动创建新方案，不再覆盖现有方案。
- 支持粘贴内容、拖放文件和选择文件导入。
- 支持复制到剪贴板或选择保存路径导出。
- 切换存在未保存修改的方案时会进行确认。
- 优化方案胶囊、删除按钮和导入导出弹窗的操作体验。
- 自定义转速改为应用时检查，编辑过程中不再自动跳回限制值。

## 设置页面

- 设置页拆分为设备、风扇和系统三个分区，结构更加清晰。
- 新增硬件概览，集中显示 CPU/GPU 温度、功耗和具体型号。
- 设备信息、实时转速和当前控制模式展示更加直观。
- 设备连接和兼容模式统一归入设备分区。
- 曲线方案、手动挡位和历史记录统一由曲线页管理，减少重复设置。
- 优化分区切换动画，进入页面保持顺序加载，切换时仅更新下方内容。
- 设置操作失败时提供明确的软件原生提醒。

## 传感器与设备控制

- 打开前台时提供完整传感器列表，确保可以正常选择所需传感器。
- 后台运行时按照用户设置读取传感器，减少不必要的硬件唤醒。
- 保持独立显卡读取模式优先，避免自动唤醒未启用的独立显卡。
- 当前温度源暂时不可用时，可使用已经读取到的另一温度源继续控制。
- 优化设备连接状态展示，连接、扫描和恢复过程更加清晰。
- 改善 WiFi、串口和百分比转速设备的界面兼容性。

## 稳定性与性能

- 改善睡眠和休眠后的设备恢复能力。
- 手动断开的设备不会在系统唤醒后被自动重新连接。
- 延长设备恢复时间窗口，提高部分设备休眠后的重连成功率。
- 降低历史数据、WiFi 扫描和传感器更新造成的前端刷新开销。
- 避免核心服务短暂重连时重复执行设备操作。
- 提高曲线方案创建和温度控制的可靠性。

## 界面体验

- 优化页面和卡片切换动画，整体响应更加紧凑流畅。
- 简化首页设备状态与温度卡片，减少重复信息。
- 改善长设备名称和硬件型号的显示效果。
- 优化兼容模式、曲线管理和设置分区的空间布局。
- 保留现有主题和毛玻璃效果的兼容性。

## 应用更新

- 优化应用内检查、下载和安装更新的操作流程。
- 下载进度通过独立更新窗口显示，不再遮挡软件界面。
- 手动检查为最新版时使用软件原生提醒。

## 下载

- `FanControl-2.5.0-amd64-installer.exe`：安装版
- `FanControl-2.5.0-portable.zip`：便携版

支持从旧版本直接升级，已有配置、设备方案、曲线、主题和历史数据会继续保留。
