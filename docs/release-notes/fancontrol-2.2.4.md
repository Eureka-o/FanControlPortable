# FanControl 2.2.4

FanControl 2.2.4 改善飞智 BS1 等 BLE/HID 设备的连接体验：点击连接后会自动查找可用设备，不再需要到高级设备页手动启用；WiFi 和虚拟串口设备仍按兼容模式手动选择。

## 更新内容

- BLE/HID 设备不再需要在高级设备页选择或启用；设置页点击「连接设备」会自动查找 HID 设备，未找到时再查找 BLE 设备。用户保存的 BLE/HID 档案会保留，并作为自动连接候选。
- 修复旧版本可能把 BS1 BLE 或 FlyDigi HID 记成手动启用设备的问题，升级后会自动整理为新的连接方式，减少重启或健康检查后的反复连接失败。
- 改善 BS1 BLE 连接稳定性：补充连接保持机制，并兼容部分设备缺少通知特征时的连接流程。
- 修复连接 FlyDigi BLE/HID 后仍按 WiFi 百分比设备显示的问题；前端状态、曲线、控制面板和托盘现在会以实际连接设备的 RPM 能力为准。
- 修复原生设备连接后曲线方案、学习偏移、自定义速度和调试模式仍落到兼容设备配置上的问题，保持不同设备之间的配置继承和隔离。
- WiFi 兼容设备连接失败时会走动态 IP 兼容式轻量查找，并在找到后更新已绑定设备的 IP；虚拟串口仍按已保存端口连接。
- 首页会继续显示硬件档案中的 GPU 型号，即使低功耗保护下暂时不轮询独显传感器。
- 高级设备页保留 WiFi、虚拟串口/COM、BLE、HID 用户设备的导入、导出和维护；只有 WiFi 与虚拟串口/COM 兼容设备提供手动启用，BLE/HID 统一交给原生连接自动遍历。
- 优化设置页和控制面板的项目结构，将连接、灯光、风扇控制、系统设置和调试区域拆分为更清晰的组件，降低后续维护 BLE/HID、WiFi 和虚拟串口逻辑时互相影响的风险。

## 升级说明

- 原有 WiFi IP、虚拟串口配置、用户设备、曲线方案和学习数据会继续保留。
- 如果之前 BS1 BLE 一直连不上，升级后请在设置页直接点击「连接设备」；不需要进入高级设备页启用 BLE/HID。
- WiFi 和虚拟串口/COM 设备仍在设置页的兼容模式中手动选择。
