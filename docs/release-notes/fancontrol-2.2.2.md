# FanControl 2.2.2

## 设备连接与显示

- 飞智（FlyDigi）BS1、BS2、BS2PRO、BS3、BS3PRO 已加入内置设备库，连接后会显示对应设备名称。
- 旧版通用 `Legacy RPM controller` 不再作为前端设备显示，仅保留为内部兼容控制路径。
- 优化短暂断联时的首页显示，减少瞬时掉线导致页面闪到“设备未连接”的情况。

FanControl 2.2.2 主要改善笔记本混合显卡环境下的功耗与温度读取体验，并优化首页显示层级。建议使用混合模式或遇到 GPU 功耗异常显示的用户更新。

## 混合模式优化

- 新增“独显低功耗保护”，默认开启。
- 在独显未工作时，FanControl 默认不会主动唤醒独显读取温度和功耗。
- 新增本机显卡档案，软件会在首次启动或档案缺失时自动识别 NVIDIA、Intel、AMD 显卡，并根据识别结果调整显示策略。
- NVIDIA 笔记本在默认低功耗保护下不会主动调用可能唤醒独显的驱动接口；需要强制读取 GPU 时，可在设置页关闭“独显低功耗保护”，温度和功耗仍走原来的传感器核心。
- 显卡档案只在启动阶段或首次有效传感器结果出现时核对，运行中不会反复改写档案，减少后台抖动。
- 自动模式下独显空闲约 30 秒后会停止读取 GPU 温度/功耗。
- 自动读取退回到更保守的触发方式：不再因为 PCManager 等进程挂在独显上就启用 GPU 传感器，而是等待实际 GPU 负载或显存活动。
- 优化后台 GPU 活动探测，降低间歇性 CPU 占用。
- GPU 未读取时，GPU 温度显示为“未读取”或 `--`，GPU 功耗显示为 `0 W`。

## 控制逻辑修复

- GPU 未读取的数据不会参与最高温计算、温升预判和学习机制。
- GPU 未读取状态下不会写入 GPU 温度/功耗历史点，避免曲线被异常数值拉乱。
- 备用温度读取逻辑不会在独显休眠时调用可能唤醒独显的 GPU 读取方式。

## 界面优化

- 首页温度、风扇速度和功耗信息的层级更清晰，减少拥挤感。
- 设置页增加独显低功耗保护开关，并说明混合模式下的显示逻辑。
- 托盘悬停提示和右键菜单会正确显示 GPU 未读取状态。
- 修复 THRM 主题下首页指标卡图标过淡或缺失的问题。

## 说明

- 如果你希望强制读取独显传感器，可以在设置页关闭“独显低功耗保护”。
- 功耗数据仍依赖系统和硬件监控接口；不同电脑可能存在可读项差异。
