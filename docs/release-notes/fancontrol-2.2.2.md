# FanControl 2.2.2

FanControl 2.2.2 是一个体验优化与稳定性修复版本。相较 2.2.1，本版本重点优化设备显示、GPU 温度/功耗显示和首页可读性，建议 2.2.x 用户更新。

## 设备连接

- 新增飞智（FlyDigi）BS1、BS2、BS2PRO、BS3、BS3PRO 内置设备档案。
- 优化设备连接后的名称显示，减少显示为旧版通用模板名称的情况。
- 优化短暂断联时的首页状态显示，减少页面频繁跳到“设备未连接”的情况。

## 温度与功耗

- 优化混合显卡环境下的 GPU 温度和功耗显示逻辑。
- 独显未读取时，GPU 温度会显示为“未读取”或 `--`，GPU 功耗显示为 `0 W`。
- GPU 未读取的数据不会参与最高温、温升预判、学习机制和历史曲线，避免影响控制判断。
- 功耗数据仍依赖系统和硬件传感器，不同电脑可能存在可读项差异。

## 界面优化

- 优化首页温度、风扇速度和功耗信息的显示层级。
- 优化托盘悬停提示和右键菜单中的 GPU 未读取状态显示。
- 修复 THRM 主题下部分首页图标过淡或缺失的问题。

## 升级说明

- 本版本不改变 WiFi 发包格式。
- 已有设备档案、曲线、学习数据和 IP 配置会继续保留。

## 2.2.2 热修复补充

- 修复手动模式或自定义速度切换后，部分 WiFi、飞智 HID/BLE 和通用串口/BLE 设备可能把目标速度显示为当前实际风速的问题。
- 优化无实时读回设备的状态显示：速度下发后会更新目标速度，实际风速会等待设备下一次状态回报。
- 新增 CPU/GPU 功耗传感器选择，遇到功耗读数异常时可在设置页手动选择正确的功耗来源。
- 优化功耗自动推荐逻辑，减少部分电脑因传感器匹配不准导致的功耗显示异常。
