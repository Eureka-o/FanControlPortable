# FanControl 2.1.6 发布说明

FanControl 2.1.6 是一个稳定性和体验修复版本，重点改善学习曲线、自动控制唤醒、功耗显示、WiFi 扫描和后台崩溃排查。

## 主要更新

- 修复温控曲线方案之间学习偏移互相影响的问题。现在每个曲线方案会保留自己的学习结果，切换方案时不再共用同一组学习偏移。
- 自动控制下，如果设备当前风速或目标风速回到 0，FanControl 会重新下发正向目标速度，减少散热器停转后不再唤醒的情况。
- 新增「温升预判」Beta，可在持续温升时提前给风扇曲线一个小幅补偿；如果系统能读取 CPU/GPU 功耗，会结合功耗辅助判断。该功能默认关闭。
- 新增 CPU/GPU 功耗显示：首页状态卡、温度与风扇历史、托盘悬停提示和托盘右键菜单都会在可读取时显示功耗；读取不到时会显示占位，不影响温度和风扇控制。
- 温度与风扇历史继续沿用原来的曲线布局，并额外叠加 CPU/GPU 功耗曲线；旧历史文件仍可继续读取。
- WiFi 查找设备时增加后台保护，同一时间只允许一个扫描任务运行，避免重复点击导致扫描状态混乱。
- WiFi「智能启停」Beta 的退出、停止和系统睡眠处理更稳，发送失败后不会卡住后续待机速度下发。
- 软件内 PawnIO 修复/更新会先尝试直接安装或更新，减少卸载后等待重启导致的驱动状态异常。
- 新增 Core 异常输出日志，便于定位后台进程启动失败或运行时崩溃。

## 使用建议

- 如果你使用多个温控曲线方案，建议更新到 2.1.6，以避免学习结果互相影响。
- 「温升预判」仍为 Beta 功能，建议先在自动控制稳定后再开启观察。
- 使用 WiFi 设备时，设备身份仍以已保存设备名称/档案为准，IP 只是可修改的连接属性。
- 升级后会继续继承原有配置、曲线方案、保存的 WiFi IP、主题、托盘设置、开机启动设置和设备档案。
