# FanControl 2.4.2

相比 2.4.1，本版重点修复 Intel 新平台 CPU 温度/功耗读取、开机自启与休眠恢复后的设备连接、BS1 蓝牙稳定性，并继续降低后台无效工作量。

## CPU 温度与功耗

- 为 Intel Core Ultra 9 275HX 等新平台增加经过校验的只读 MSR 兜底；当 LibreHardwareMonitor 未返回 CPU 传感器时，仍可读取 CPU 封装温度与功耗。
- 区分 PawnIO 未安装与临时句柄竞争、读取失败。临时故障会按冷却策略重试，不再一次失败后永久停止恢复，也不会持续重启桥接进程。
- 改善温度监控在界面启动、关闭和系统恢复时的生命周期，避免旧采样循环影响新一轮监控。

## 设备连接

- 开机自启和设备晚到场景会在核心服务中继续进行有界重试，无需打开界面后手动连接。
- 原生 HID/BLE 设备命中后立即连接；只有原生设备未命中时才继续 WiFi 兼容扫描，缩短常用连接路径。
- 修复休眠/唤醒时旧重连任务覆盖新连接的问题。手动断开后，已经开始的旧连接任务也不会把设备重新连回。
- 改善 BS1 的蓝牙扫描、连接、GATT 探测和健康检查仲裁；失效连接会被清理，后续读取可重新建立连接。
- 改善 GUI 与核心服务重连时的请求和事件代次隔离，旧连接不会清理或覆盖新连接状态。

## 智能温控与异常遥测

- CPU 功耗暂时无法读取时，会保持为“未知”并采用保守的温度/转速学习，不会误当作 `0W` 影响风扇曲线。
- 异常、缓存或过期的温度数据不会参与温升预判和学习偏移；基础自动控速仍会继续工作，优先保证散热安全。
- 温升预判会在持续升温或有效功耗快速增加时临时加快升速；当风扇实际转速尚未跟上目标时，不会继续叠加预判。
- 静音方向的学习需要连续稳定确认，高温时的安全升速不等待确认，减少不必要的降速与反复修正。

## 后台与流畅度

- 合并启动阶段的重复配置保存，并使用同目录临时文件完成配置替换，降低写入次数和中断写入风险。
- 减少重复的前端配置回写、设备方案请求、窗口尺寸查询、调试帧分配和非调试日志写入。
- WiFi 深度扫描按实际并发量分配结果通道，避免按全部候选地址预分配。
- 移除未使用的前端组件与依赖，降低安装依赖和构建体积，不增加新的后台轮询。

## 下载

- `FanControl-2.4.2-amd64-installer.exe`：安装版，推荐大多数用户使用。
- `FanControl-2.4.2-portable.zip`：便携版，解压后运行 `FanControl.exe`。
