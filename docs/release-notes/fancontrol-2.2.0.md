# FanControl 2.2.0 更新说明

FanControl 2.2.0 重点改进多设备使用体验、设备设置结构和温度桥接稳定性。建议所有 2.0 / 2.1 用户更新。

## 设备与曲线

- 曲线方案现在按设备独立保存。切换设备后会自动加载该设备自己的曲线，不需要再点一次保存才能让智能曲线生效。
- 学习结果按“设备 + 曲线方案”隔离，同名曲线不会在 WiFi、HID、RPM 或百分比设备之间串用学习偏移。
- 连接或切换设备后，FanControl 会重新匹配当前设备档案、速度单位和设备能力，减少设置项、曲线单位和连接信息互相串台。
- 首页、标题栏、托盘悬停提示和右键菜单会跟随当前设备速度单位显示风速：百分比设备显示 `%`，RPM 设备显示 `RPM`。

## 设备设置与高级设备

- 设置页新增集中的「设备设置」区域，当前设备支持的基础功能、智能启停和灯光控制会集中显示。
- 设备功能继续跟随能力白名单：设备没有声明支持的功能不会在普通设置页出现，后端也会继续拒绝不支持的调用。
- 高级设备配置页重新分组，设备身份、连接方式、速度单位、WiFi/BLE/虚拟串口参数、协议模板和解析规则更容易维护。
- 智能启停设置合并为单层设置行，WiFi Beta 待机策略只在开启后展开待机速度子项。

## 稳定性与诊断

- 温度桥接程序改为独立子进程 stdio 通信，减少硬件传感器读取偶发卡住时导致的“桥接程序通信失败 / i/o timeout”。
- 温度读取请求使用更宽松的专用超时；失败后会主动丢弃不可信的桥接进程，下次读取会自动重新启动。
- TempBridge 启动和硬件监控初始化会记录更清晰的诊断日志，方便排查 PawnIO、LibreHardwareMonitor 或传感器枚举卡住的问题。
- 错误提示和温度桥接异常提示增加“导出诊断日志”，反馈问题时可以保存 FanControl 专属诊断包。

## 界面与体验

- Windows 11 窗口增加 Mica 毛玻璃效果，并同步系统深浅色主题，使主界面和标题栏更接近系统原生窗口。
- WiFi「查找 IP」前端逻辑拆成独立模块，普通查找、深度扫描、暂停、中止和用时显示保持原有体验，为后续动态 IP 搜索和断线重连继续做准备。
- 首页工作模式只显示「自动模式」或「手动模式」，不再把 WiFi、BLE、HID 或协议调试文本当成工作模式。
- 修复 FlyDigi HID 控制包写入：短控制报告会按 Windows HID 输出报告长度补齐后发送，减少“能连接但速度不变化”的情况。
- 自定义速度和设备功能指令下发失败时会给出错误提示，不再静默吞掉失败状态。

## 升级说明

- 本版本不改变 WiFi 发包格式。
- 不删除已保存设备，也不会重置主题、托盘设置或开机启动设置。
- 不建议同时运行多个会控制同一散热器的软件；如果两个软件同时向同一个设备下发速度，设备会以最后收到的指令为准，可能出现速度来回变化。
- 升级后会继续保留原有配置、曲线方案、保存的 WiFi IP、主题、托盘设置、开机启动设置和设备档案。
- 如果你只使用默认 WiFi 散热器，可以直接升级，不需要重新配置设备。
