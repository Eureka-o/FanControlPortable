# FanControl 2.2.0 更新说明

FanControl 2.2.0 主要整理设备功能与高级设备配置，让后续适配更多设备时更清楚、更好维护。

## 主要变化

- 设置页新增集中的「设备设置」区域，会把当前设备支持的基础功能、智能启停和灯光控制分成小分区显示。
- 设备功能继续跟随设备能力白名单：设备没有声明支持的功能不会在普通设置页中出现，后端也会继续拒绝不支持的调用。
- 高级设备配置页增加更清晰的分组说明，便于维护设备身份、连接方式、速度单位、WiFi/BLE/虚拟串口参数、协议模板和解析规则。
- WiFi「查找 IP」前端逻辑拆成独立模块，普通查找、深度扫描、暂停、中止和用时显示保持原有体验，为后续动态 IP 搜索和断线重连继续做准备。
- 首页连接状态会跟随当前设备的实际连接方式显示，工作模式只显示「自动模式」或「手动模式」，不再把 WiFi、BLE、HID 或协议调试文本当成工作模式。
- 托盘悬停提示和右键菜单会跟随当前设备速度单位显示风速：百分比设备显示 `%`，RPM 设备显示 `RPM`，避免转速设备被误显示为百分比。
- 连接或切换设备后会重新匹配当前设备档案与能力，减少不同设备之间的设置项、速度单位和连接信息互相串台。
- 曲线方案现在会按设备独立保存。切换设备后会自动加载该设备自己的曲线，不需要再点一次保存才能让智能曲线生效。
- 学习结果会按“设备 + 曲线方案”隔离，同名曲线不会在不同设备之间串用学习偏移。
- 修复 FlyDigi HID 控制包写入：短控制报告会按 Windows HID 输出报告长度补齐后发送，更接近参考软件的 hidapi 写入行为，减少“能连接但速度不变化”的情况。
- 自定义速度和设备功能指令下发失败时会给出错误提示，不再静默吞掉失败状态。
- 智能启停设置合并为单层设置行，WiFi Beta 待机策略只在开启后展开待机速度子项，界面更接近其它设置项。

## 升级说明

- 本版本不改变 WiFi 发包格式。
- 不建议同时运行多个会控制同一散热器的软件；如果两个软件同时向同一个设备下发速度，设备会以最后收到的指令为准，可能出现速度来回变化。
- 升级后会继续保留原有配置、曲线方案、保存的 WiFi IP、主题、托盘设置、开机启动设置和设备档案。
- 如果你只使用默认 WiFi 散热器，可以直接升级，不需要重新配置设备。
