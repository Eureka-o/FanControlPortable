# FanControl

FanControl 是面向 Windows 的散热控制软件，用于通过设备档案连接和控制多种散热器、风扇控制器和 DIY 主控。它可以读取电脑 CPU/GPU 温度，显示设备状态，并根据手动速度或风扇曲线控制散热设备；当前内置默认设备为 **Slim压风散热器Pro**。

[下载最新版](https://github.com/Eureka-o/FanControlPortable/releases/latest)

## 适用范围

- 系统：Windows 10 / Windows 11 64 位。
- 内置默认设备：Slim压风散热器Pro；同时提供 WiFi、BLE、虚拟串口/COM 和旧版 HID/RPM 的设备档案基础。
- 默认连接方式：WiFi。
- 高级设备页：FanControl 2.1.0 增加了面向 DIY 主控的 WiFi、BLE、虚拟串口/COM 和旧版 HID/RPM 档案基础；普通用户不需要进入高级设备页也可以正常使用默认设备。

## 下载哪个文件

- `FanControl-2.2.0-amd64-installer.exe`：推荐大多数用户使用。安装后可以从开始菜单或桌面启动，升级 2.0/2.1 到 2.2.0 时会保留已有配置。
- `FanControl-2.2.0-portable.zip`：免安装便携版。解压到一个固定文件夹后运行 `FanControl.exe`。

首次启动时如果 Windows 弹出权限确认，请选择允许。软件需要管理员权限读取硬件温度，并可能安装或调用温度读取所需的辅助组件。

## 第一次使用

1. 确认电脑和散热器处在同一个网络环境中。
2. 打开 FanControl。
3. 进入设置页，连接方式选择 `WIFI`。
4. 填写设备 IP 地址。常见默认地址为 `192.168.137.2`，也支持 `127.0.0.1:18080` 这种带端口的写法；如果设备已经保存为 WiFi 档案，也可以在设置页使用「查找 IP」。
5. 点击连接，回到主页查看温度、当前风速、目标风速和控制模式。

连接成功后，可以使用自动曲线控制，也可以切换到手动速度进行固定百分比控制。

## 主要功能

- 实时查看 CPU/GPU 温度和散热器风速。
- 自动模式下按风扇曲线调节速度。
- 手动模式下直接设定固定百分比速度。
- 百分比控制和学习机制支持 `0.1%` 内部精度，并在发包时按设备能力取整。
- 查看温度、功耗和风扇速度历史记录。
- 最小化到系统托盘后后台运行。
- 从软件内检查 GitHub 最新版本。
- 安装版和便携版都会尽量保留已有配置。

## 2.2.0 更新内容

- 设置页将设备相关功能集中到同一个「设备设置」区域，并按基础功能、智能启停、灯光控制分成小分区，显示逻辑继续跟随当前启用设备的能力白名单。
- 高级设备配置页增加更清晰的分组说明：基础设备信息、设备功能白名单、WiFi/BLE/虚拟串口连接配置、协议与发包模板会分区显示，方便后续维护 DIY 设备档案。
- WiFi「查找 IP」前端逻辑拆成独立模块，普通查找、深度扫描、暂停、中止和用时显示保持原有行为，为后续动态 IP 搜索和断线重连继续做准备。
- 首页、标题栏和设置页会根据当前设备重新匹配连接方式、速度单位和设备能力；首页工作模式只显示「自动模式」或「手动模式」，不再显示设备类型或协议调试文本。
- 托盘悬停提示和右键菜单会跟随当前设备速度单位显示风速，百分比设备显示 `%`，RPM 设备显示 `RPM`。
- 曲线方案现在会按设备独立保存。切换设备后会自动加载该设备自己的曲线，不需要再点一次保存才能让智能曲线生效。
- 学习结果会按“设备 + 曲线方案”隔离，同名曲线不会在不同设备之间串用学习偏移。
- 修复 FlyDigi HID 控制包写入：短控制报告会按 Windows HID 输出报告长度补齐后发送，减少“能连接但无法改变速度”的情况；自定义速度和设备功能指令下发失败时会直接提示原因。
- 智能启停设置改为单层设置行，WiFi Beta 待机速度只在启用后作为子项展开。
- 如果同时运行多个会控制同一个散热器的软件，设备会以最后收到的速度指令为准，可能导致控制结果来回变化；建议同一时间只保留一个控制软件接管设备。
- 本版本不改变 WiFi 发包格式，不删除已保存设备，也不会影响已有配置、曲线方案、主题、托盘设置、开机启动设置和设备档案。

## 2.1.7 修复内容

- 改进温度桥接程序稳定性：FanControl 现在会以独立子进程方式通过 stdio 管理 TempBridge，减少硬件传感器读取偶发卡住时导致的“桥接程序通信失败 / i/o timeout”。
- 温度读取请求使用更宽松的专用超时，并在失败后主动丢弃不可信的桥接进程，下次读取会自动重新启动桥接程序。
- TempBridge 启动和硬件监控初始化会记录更清晰的诊断日志，方便排查 PawnIO、LibreHardwareMonitor 或传感器枚举卡住的问题。
- 错误提示和温度桥接异常提示增加“导出诊断日志”，会保存 FanControl 专属诊断包，方便反馈问题时一次性提供日志、桥接状态和配置摘要。
- 保留 2.1.6 的 CPU/GPU 功耗读取、历史记录和托盘显示能力，不影响已有温度、功耗和风扇控制逻辑。
- Windows 11 窗口增加 Mica 毛玻璃效果，并同步系统深浅色主题，使主界面和标题栏视觉更贴近系统原生窗口。
- 升级后会继续继承原有配置、曲线方案、保存的 WiFi IP、主题、托盘设置、开机启动设置和设备档案。

## 2.1.6 修复内容

- 修复温控曲线方案之间学习偏移互相影响的问题。切换曲线方案时，每个方案会保留自己的学习结果。
- 自动控制下如果设备当前风速或目标风速回到 0，FanControl 会重新下发正向目标速度，减少散热器停转后不再唤醒的情况。
- 新增「温升预判」Beta：可根据持续温升趋势，并在可用时结合 CPU/GPU 功耗，提前给风扇曲线一个小幅补偿；默认关闭。
- 新增 CPU/GPU 功耗显示：首页状态卡、温度与风扇历史、托盘悬停提示和托盘右键菜单都会在可读取时显示功耗；读取不到时会显示占位，不影响温度和风扇控制。
- 温度与风扇历史继续沿用原来的曲线布局，并额外叠加 CPU/GPU 功耗曲线；旧历史文件仍可继续读取。
- 优化 WiFi 扫描后台保护：同一时间只允许一个扫描任务运行，避免重复点击导致扫描状态混乱。
- 优化 WiFi「智能启停」Beta：退出、停止和系统睡眠前的待机速度下发更稳，发送失败后不会卡住后续处理。
- 优化 PawnIO 修复动作：软件内修复/更新会先尝试直接安装或更新，减少卸载后等待重启导致的驱动状态异常。
- 新增 Core 异常输出日志，便于排查后台进程启动或运行时崩溃。
- 升级后会继续继承原有配置、曲线方案、保存的 WiFi IP、主题、托盘设置、开机启动设置和设备档案。

## 2.1.3 新增内容

- 新增 WiFi「查找 IP」：基于已保存的 WiFi 设备档案查找散热器地址，设备未连接时也可以使用。
- 新增「深度扫描」：普通查找无结果后显示，继续检查常见路由器网段和设备热点网段。

- 继续保留 BLE/HID 连接方式给已导入或手动配置的设备；未完成验证的飞智设备信息暂不在正式界面开放。

## 使用提示

- 关闭窗口不等于退出软件。FanControl 会继续留在系统托盘中运行；需要完全退出时，请右键托盘图标并选择退出。
- 如果同时使用其他硬件监控或风扇控制软件，遇到温度读取异常时可以先临时关闭它们再重试。
- 高级设备页适合已经了解主控协议的用户。原始调试命令可能影响设备状态，请只在确认命令含义后发送。
- BLE 和虚拟串口/COM 的通用档案能力已经具备基础运行时，但不同 DIY 设备仍需要按主控协议填写正确字段；真实硬件兼容性需要设备用户继续反馈。

## 常见问题

### 设备连接不上

- 确认散热器已通电，并且电脑和散热器处于同一网络。
- 检查设置页中的设备 IP 地址是否正确。
- 如果修改过散热器网络设置，请重新填写新的 IP 地址。
- 可以在浏览器中打开 `http://设备地址/api/data` 做简单确认；能看到返回内容通常说明网络连通。

### 温度为空或一直是 0

- 以管理员身份重新启动 FanControl。
- 安装器提示安装或更新温度读取组件时，请允许。
- 临时关闭其他硬件监控软件后再试一次。
- 如果便携版缺少 `bridge` 目录，请重新下载并完整解压便携包。

### 风速没有按预期变化

- 确认设备已经连接成功。
- 检查当前是自动模式还是手动模式。
- 手动模式会固定在你设定的百分比速度；自动模式会按风扇曲线计算目标速度。
- 速度下发后设备状态可能需要几秒钟刷新。
